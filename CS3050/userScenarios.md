# [User Senarios](https://github.com/Hanif-K-Musaheb/Year-3-CS-UVM-Study-Abroad-/blob/main/CS3050/CS3050home.md)
#### Basic Description
A user scenario is a narrative or story that describes how a particular user (or type of user) interacts with a software system to achieve a specific goal.
### It focuses on:
- **Who** the user is?
- **What** they want to do?
- **Why** they want to do it?
- **How** they interact with the system to achieve it?

  
| **Stage**                    | **How Scenarios Are Used**                                               |
| ---------------------------- | ------------------------------------------------------------------------ |
| **Requirements Analysis**    | To define user needs and functional requirements clearly                 |
| **System Design**            | To design workflows, UI elements, and navigation that fit real use cases |
| **Prototyping**              | To test and validate early designs against realistic situations          |
| **Testing & Evaluation**     | To create realistic test cases based on user behavior                    |
| **Training & Documentation** | To show end-users how the system supports their real-world tasks         |
